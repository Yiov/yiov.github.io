# DocSearch

更新时间

最近更新：2023-9-12

搭建版本：v2.0-beta.67

# 安装

说明

安装失败，挂个梯子就可以了

#这里是pnpm 新增@algolia/client-search
pnpm add -D @vuepress/plugin-docsearch@next @algolia/client-search search-insights

#这里是yarn 新增@algolia/client-search
yarn add -D @vuepress/plugin-docsearch@next @algolia/client-search search-insights

#这里是npm 新增@algolia/client-search
npm i -D @vuepress/plugin-docsearch@next @algolia/client-search search-insights

# 配置

然后在ts里添加配置

注意

appId / apiKey / indexName 这3个我们等会获取了再填入

import { docsearchPlugin } from '@vuepress/plugin-docsearch'

export default {
  //========docsearch配置========//
  plugins: [
    docsearchPlugin({
      appId: '<Application ID>',
      apiKey: '<Search-Only API Key>',
      indexName: '<INDEX_NAME>',
      locales: {
        '/': {
          placeholder: '搜索文档',
          translations: {
            button: {
              buttonText: '搜索文档',
            },
          },
        },
        '/en/': {
          placeholder: 'Search Documentation',
          translations: {
            button: {
              buttonText: 'Search Documentation',
            },
          },
        },
      },
    }),
  ],
}

# 使用

在注册账号前，请先确定自己使用 自行申请 还是 自行爬取

自行申请：可使用algolia官方的爬虫，但是需要提交公开仓库代码链接

自行爬取：需有一定动手能力，自行搭建爬虫，不需要提交仓库代码链接

# Github Actions爬取数据

推荐

这个和 Docker爬取数据 任选其一，推荐此方式

我们先创建一个工作流

说明

我因为已经有一个了，我直接新建一个就行

命名为docsearch.yml ，复制粘贴下面的工作流代码，提交

on: deployment

on:
  push:
    branches:
      - main

on:
  schedule:
    # 约每天早上8点触发（UTC时间0点）
    - cron: '0 0 * * *'

on:
  workflow_dispatch:

注意

这里触发的条件自己选，改代码高亮的位置就行，其他不要动

# 名字可以自己取
name: docsearch

# 提交main分支触发部署
on:
  push:
    branches:
      - main

jobs:
  algolia:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      # 一会要建docsearch.json文件
      - name: Get the content of docsearch.json as config
        id: algolia_config
        run: echo "::set-output name=config::$(cat docsearch.json | jq -r tostring)"

      - name: Run algolia/docsearch-scraper image
        # 环境变量 ，在仓库设置-安全里添加秘钥
        # APPLICATION_ID 是 algoia 的 APPLICATION ID
        # API_KEY 是 algolia 的 Admin API KEY
        # CONFIG默认即可，无需更改
        env:
          APPLICATION_ID: ${{ secrets.APPLICATION_ID }}
          API_KEY: ${{ secrets.API_KEY }}
          CONFIG: ${{ steps.algolia_config.outputs.config }}
        run: |
          docker run \
            --env APPLICATION_ID=${APPLICATION_ID} \
            --env API_KEY=${API_KEY} \
            --env "CONFIG=${CONFIG}" \
            algolia/docsearch-scraper
