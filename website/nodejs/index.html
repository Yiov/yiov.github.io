# node.js安装使用

更新时间

最近更新：2023-2-23

node.js是JavaScript的运行环境，本次仅演示 linux 和 window 安装方式

说明

安装了宝塔的可直接在软件商店搜node安装

  • 官网：https://nodejs.org/zh-cn/open in new window

  • 仓库：https://github.com/nodejs/nodeopen in new window

# linux系统安装

选择 linux x64下载，为了保证稳定性，建议选择长期维护的版本

不用直接点击下载，我们用命令

# 切换root账号

一般都是root，多加一步不影响

鼠标右键粘贴！ctrl+v是不可用的

sudo su root #切换到root帐号
cd / #进入根目录

# 下载解压安装

版本我用的最新的V14.17.4，可按照官网最新的版本改好命令，保持一致

说明

其他版本查找：https://nodejs.org/dist/open in new window

依次运行命令

#下载nodejs安装包
wget https://nodejs.org/dist/v14.17.4/node-v14.17.4-linux-x64.tar.xz

#解压
tar xvf node-v14.17.4-linux-x64.tar.xz

#改名
mv node-v14.17.4-linux-x64 node-v14.17.4

等待完成，停在 改名 那 回车 一下就行了

进目录查看文件夹，有就代表安装好了

注意版本号一致

cd /node-v14.17.4/bin && ls #进入目录并列出

# 建立软连接

注意版本号一致

ln -s /node-v14.17.4/bin/node /usr/local/bin/node
ln -s /node-v14.17.4/bin/npm /usr/local/bin/npm

# 重启Xshell

完成后一定要关掉 Xshell 再打开，否则不生效

# 查看版本

确定重启后，复制下面命令查看一下

cd /node-v14.17.4/bin && ls
node -v #查看版本号
npm -v #查看包管理工具

# 清理安装包(可选)

查看目录列表，标红的node***.tar.xz就是刚下的压缩包

cd /
ls

删除node安装包，复制下列命令

说明

版本不同的话，自己改下名

我这里有两个，是因为我运行了2次，下载重复了

rm -rf node-v14.17.4-linux-x64.tar.xz

删完后，我们再进目录看一下，完美

cd /
ls

# Window安装

# 安装

官网下载我们选择window的方式

下载好安装，就不用说了吧

安装好后，我们运行 window键+R 进入 cmd 终端

查看一下版本号，有就说明成功了

node -v
npm -v

# 更改全局模块(可选)

默认情况下，下载yarn或者pnpm的时候就会直接下载到C盘了

说明

如果想挪动到安装目录的就看，无所谓的无视，不影响使用

在安装目录下创建两个文件夹 node_global 和 node_cache

打开cmd 输入下列指令

说明

全程请参照自己的安装目录来修改！！！

prefix = 创建的node_global文件夹所在路径

cache = 创建的node_cache文件夹所在路径

npm config set prefix "D:\Program Files\nodejs\node_global"
npm config set cache "D:\Program Files\nodejs\node_cache"

修改系统环境变量，将 用户变量 下的 Path 修改为如下，之后点击确定

D:\Program Files\nodejs\node_global

说明

此电脑-右键-属性-高级系统设置-环境变量

在 系统变量 下新建 NODE_PATH ，值如下

D:\Program Files\nodejs\node_global\node_modules

在 系统变量 下的 Path 新建，添加node全局文件夹之后点击确定

D:\Program Files\nodejs\node_global

右击Nodejs文件夹-属性-安全，点击编辑，将所有权限都✔上

说明

关掉cmd重新打开，看是否能查询版本

node -v
npm -v

# Yarn和Pnpm

我们常用的yarn和pnpm可以安装一下，顺便测试一下全局模块是否正确

#安装pnpm
npm install -g pnpm
#查看版本号
pnpm -v

#安装yarn
npm install -g yarn
#查看版本号
yarn -v

win11上居然是报错，是因为power shell的安全策略，不允许执行不安全脚本

nrm : 无法加载文件 C:\Users\用户\AppData\Roaming\npm\nrm.ps1，因为在此系统上禁止运行脚本

搜索 power shell 或者 power shell ISE ，管理员方式打开，输入如下值，全是A

说明

默认值：set-ExecutionPolicy Default

set-ExecutionPolicy RemoteSigned
