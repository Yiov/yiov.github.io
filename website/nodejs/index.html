# linux系统安装

选择 linux x64下载，为了保证稳定性，建议选择长期维护的版本

说明

不用直接点击下载，我们用命令

切换root账号

说明

一般都是root，多加一步不影响

鼠标右键粘贴！ctrl+v是不可用的

sudo su root #切换到root帐号
cd / #进入根目录

下载安装，版本是目前最新的V14.17.4，可按照官网最新的版本改好命令，保持一致

说明

其他版本查找：https://nodejs.org/dist/

依次运行命令

#下载nodejs安装包
wget https://nodejs.org/dist/v14.17.4/node-v14.17.4-linux-x64.tar.xz

#解压
tar xvf node-v14.17.4-linux-x64.tar.xz

#改名
mv node-v14.17.4-linux-x64 node-v14.17.4

等待完成，停在 改名 那 回车 一下就行了

进目录查看文件夹，有就代表安装好了

说明

注意版本号一致

#进入目录并列出
cd /node-v14.17.4/bin && ls

建立软连接

说明

注意版本号一致

ln -s /node-v14.17.4/bin/node /usr/local/bin/node
ln -s /node-v14.17.4/bin/npm /usr/local/bin/npm

重启Xshell

说明

完成后一定要关掉 Xshell 再打开，否则不生效

确定重启后，复制下面命令查看一下

cd /node-v14.17.4/bin && ls
node -v #查看版本号
npm -v #查看包管理工具

清理安装包，查看目录列表，标红的 node***.tar.xz 就是刚下的压缩包

cd /

删除node安装包，复制下列命令

版本不同的话，自己改下名

# 更改全局模块(可选)

说明

默认情况下，下载 yarn 或者 pnpm 会直接下载到 C盘

如果想挪到安装目录的就看，无所谓的无视，不影响使用

在安装目录下创建两个文件夹 node_global 和 node_cache

打开cmd 输入下列指令

说明

全程请参照自己的安装目录来修改！！！

prefix = 创建的node_global文件夹所在路径

cache = 创建的node_cache文件夹所在路径

npm config set prefix "D:\Program Files\nodejs\node_global"
npm config set cache "D:\Program Files\nodejs\node_cache"

修改系统环境变量，将 用户变量 下的 Path 修改为如下，之后点击确定

D:\Program Files\nodejs\node_global

说明

此电脑-右键-属性-高级系统设置-环境变量

在 系统变量 下新建 NODE_PATH ，值如下

D:\Program Files\nodejs\node_global\node_modules

在 系统变量 下的 Path 新建，添加node全局文件夹之后点击确定

D:\Program Files\nodejs\node_global

右击Nodejs文件夹-属性-安全，点击编辑，将所有权限都✔上

说明

关掉cmd重新打开，看是否能查询版本

node -v
npm -v

# YARN 和 PNPM

我们常用的 yarn和 pnpm 可以安装一下，顺便测试一下全局模块是否正确

npm install -g pnpm
#查看版本号
pnpm -v

#安装yarn
npm install -g yarn
#查看版本号
yarn -v

说明

win11上如果报错，是因为power shell的安全策略，不允许执行不安全脚本

nrm : 无法加载文件 C:\Users\用户\AppData\Roaming\npm\nrm.ps1，因为在此系统上禁止运行脚本

解决：搜索 power shell 或者 power shell ISE ，管理员方式打开，输入如下值，然后全选A

set-ExecutionPolicy RemoteSigned

想要改回来

set-ExecutionPolicy Default

# iOS安装

我们要用到 iSH shell ，它是一款iOS端终端 Alpine Linux x86系统，采用 musl libc 和 busybox 构建 ，自带apk管理软件包

说明

apk 是 alpine linux 中的包管理器，类似 ubuntu 中的 apt，centos 中的yum

由于IOS系统特殊性，我们只能使用npm，无法升级成yarn/pnpm

# 安装iSH

在苹果商店下载安装并打开

https://apps.apple.com/cn/app/id1436902243

# 升级apk

更新镜像源和软件包

#更新源索引
apk update

#更新软件包
apk upgrade

# 安装nodejs

node和npm要分别安装

注意

由于系统默认是v3.14，最新的 Alpinelinux 中的 nodejs 是 14.21.3 ，而pmpm 需要16以上

无法使用 npm install -g yarn或者pnpm命令，只能使用npm

#安装
apk add nodejs

#查询版本
node -v

#安装
apk add npm

#查询版本
npm -v

# 4.linux如何换源

默认一般都是国外源，所以下载软件包的时候会有点慢，想快的就需要换国内镜像源

  • 阿里云源：https://developer.aliyun.com/mirror/

  • 华为云源：https://mirrors.huaweicloud.com/

  • 兰州大学源：https://mirror.lzu.edu.cn/

  • 清华大学源：https://mirrors.tuna.tsinghua.edu.cn/

  • 上海交大源：https://mirror.sjtu.edu.cn/

  • 腾讯源：https://mirrors.cloud.tencent.com/

  • 网易源：https://mirrors.163.com/

  • 中科大USTC：https://mirrors.ustc.edu.cn/

那么我们的npm就可以换源了

说明

其他源自行修改后面的源地址

npm config set registry http://mirrors.cloud.tencent.com/npm/

除了常见的用 vim或nano 编辑器 修改换源，还可以使用一键脚本 sed -i 命令

示例：

Centos 换成 清华大学源

# 对于 CentOS 7
sudo sed -e 's|^mirrorlist=|#mirrorlist=|g' \
         -e 's|^#baseurl=http://mirror.centos.org/centos|baseurl=https://mirrors.tuna.tsinghua.edu.cn/centos|g' \
         -i.bak \
         /etc/yum.repos.d/CentOS-*.repo

# 对于 CentOS 8
sudo sed -e 's|^mirrorlist=|#mirrorlist=|g' \
         -e 's|^#baseurl=http://mirror.centos.org/$contentdir|baseurl=https://mirrors.tuna.tsinghua.edu.cn/centos|g' \
         -i.bak \
         /etc/yum.repos.d/CentOS-*.repo

Ubuntu 换成 兰州大学源

sudo sed -i "s@http://.*archive.ubuntu.com@https://mirror.lzu.edu.cn@g" /etc/apt/sources.list
sudo sed -i "s@http://.*security.ubuntu.com@https://mirror.lzu.edu.cn@g" /etc/apt/sources.list

Debian 换成 中科大源

sodo sed -i "s@http://deb.debian.org@https://mirrors.ustc.edu.cn/g" /etc/apt/sources.list
sudo sed -i 's/deb.debian.org/mirrors.ustc.edu.cn/g' /etc/apt/sources.list
